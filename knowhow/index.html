🎨 フロントエンド開発

Frontend Development - User Interface & Experience

🇯🇵 日本語

モダンフロントエンド開発の基礎

現代のWebアプリケーション開発において、フロントエンドは単なる見た目の装飾から、複雑なビジネスロジックを含む重要な役割を担うようになりました。React、Vue.js、Angular などのフレームワークを活用し、コンポーネント指向の設計で保守性と再利用性を高めることが重要です。

特に重要なのは、レスポンシブデザインの実装、パフォーマンスの最適化、アクセシビリティの確保です。CSS-in-JSやSass/SCSSを使用したスタイル管理、WebpackやViteなどのバンドラーを使った効率的なビルドプロセスの構築も必須のスキルとなっています。

🇺🇸 English

Modern Frontend Development Fundamentals

In modern web application development, frontend has evolved from mere visual decoration to handling complex business logic. Utilizing frameworks like React, Vue.js, and Angular with component-oriented design is crucial for maintainability and reusability.

Key focuses include responsive design implementation, performance optimization, and accessibility compliance. Skills in CSS-in-JS or Sass/SCSS for style management, and efficient build processes using bundlers like Webpack or Vite are also essential.

React コンポーネントの例 / React Component Example JSX 
import React, { useState, useEffect } from 'react';
import './UserProfile.css';

const UserProfile = ({ userId }) => {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    const fetchUser = async () => {
      try {
        setLoading(true);
        const response = await fetch(`/api/users/${userId}`);
        if (!response.ok) {
          throw new Error('ユーザー情報の取得に失敗しました / Failed to fetch user data');
        }
        const userData = await response.json();
        setUser(userData);
      } catch (err) {
        setError(err.message);
      } finally {
        setLoading(false);
      }
    };

    if (userId) {
      fetchUser();
    }
  }, [userId]);

  if (loading) {
    return (
      <div className="user-profile-loading">
        <div className="spinner" aria-label="読み込み中 / Loading..."></div>
      </div>
    );
  }

  if (error) {
    return (
      <div className="user-profile-error" role="alert">
        <h3>エラー / Error</h3>
        <p>{error}</p>
        <button onClick={() => window.location.reload()}>
          再試行 / Retry
        </button>
      </div>
    );
  }

  return (
    <div className="user-profile" itemScope itemType="https://schema.org/Person">
      <div className="user-avatar">
        <img 
          src={user.avatar || '/default-avatar.png'} 
          alt={`${user.name}のアバター / ${user.name}'s avatar`}
          itemProp="image"
        />
      </div>
      <div className="user-info">
        <h2 itemProp="name">{user.name}</h2>
        <p className="user-title" itemProp="jobTitle">{user.title}</p>
        <p className="user-bio" itemProp="description">{user.bio}</p>
        <div className="user-stats">
          <div className="stat">
            <span className="stat-label">投稿数 / Posts</span>
            <span className="stat-value">{user.postsCount}</span>
          </div>
          <div className="stat">
            <span className="stat-label">フォロワー / Followers</span>
            <span className="stat-value">{user.followersCount}</span>
          </div>
        </div>
      </div>
    </div>
  );
};

export default UserProfile;

CSS設計手法 / CSS Architecture Methodologies ▼

BEM (Block Element Modifier)

BEMは最も人気の高いCSS設計手法の一つで、クラス名の命名規則を統一することで保守性を向上させます。

  • Block: 独立したコンポーネント (.header, .menu, .button)
  • Element: ブロックの構成要素 (.header__title, .menu__item)
  • Modifier: 状態や変種 (.button--large, .menu__item--active)

BEM (Block Element Modifier)

BEM is one of the most popular CSS architecture methodologies that improves maintainability through consistent class naming conventions.

  • Block: Independent component (.header, .menu, .button)
  • Element: Block's constituent parts (.header__title, .menu__item)
  • Modifier: State or variation (.button--large, .menu__item--active)

⚙️ バックエンド開発

Backend Development - Server-side Architecture & APIs

🇯🇵 日本語

スケーラブルなサーバーアーキテクチャ

モダンなバックエンド開発では、マイクロサービスアーキテクチャ、RESTful API設計、GraphQLの活用が重要です。Node.js、Python（Django/Flask）、Go、Rustなどの言語を適材適所で選択し、データベース設計からキャッシング戦略まで総合的に考える必要があります。

特にクラウドネイティブな開発では、Docker コンテナ化、Kubernetes でのオーケストレーション、CI/CD パイプラインの構築が必須となっています。セキュリティ対策、パフォーマンス最適化、監視・ログ管理も含めた包括的な設計が求められます。

🇺🇸 English

Scalable Server Architecture

Modern backend development emphasizes microservices architecture, RESTful API design, and GraphQL utilization. Choosing appropriate technologies like Node.js, Python (Django/Flask), Go, or Rust, and comprehensive consideration from database design to caching strategies is essential.

In cloud-native development, Docker containerization, Kubernetes orchestration, and CI/CD pipeline construction are mandatory. Comprehensive design including security measures, performance optimization, and monitoring/logging management is required.

Express.js API エンドポイントの例 / Express.js API Endpoint Example JavaScript 
const express = require('express');
const rateLimit = require('express-rate-limit');
const helmet = require('helmet');
const cors = require('cors');
const jwt = require('jsonwebtoken');
const { body, validationResult } = require('express-validator');

const app = express();

// セキュリティミドルウェア / Security Middleware
app.use(helmet());
app.use(cors({
  origin: process.env.FRONTEND_URL,
  credentials: true
}));

// レート制限 / Rate Limiting
const apiLimiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15分 / 15 minutes
  max: 100, // 最大100リクエスト / max 100 requests
  message: {
    error: 'リクエスト制限に達しました / Rate limit exceeded',
    retryAfter: '15分後に再試行してください / Please try again after 15 minutes'
  }
});

app.use('/api/', apiLimiter);
app.use(express.json({ limit: '10mb' }));

// JWT認証ミドルウェア / JWT Authentication Middleware
const authenticateToken = (req, res, next) => {
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1];

  if (!token) {
    return res.status(401).json({
      error: 'アクセストークンが必要です / Access token required'
    });
  }

  jwt.verify(token, process.env.JWT_SECRET, (err, user) => {
    if (err) {
      return res.status(403).json({
        error: '無効なトークンです / Invalid token'
      });
    }
    req.user = user;
    next();
  });
};

// バリデーションルール / Validation Rules
const createUserValidation = [
  body('email')
    .isEmail()
    .withMessage('有効なメールアドレスを入力してください / Please provide a valid email address')
    .normalizeEmail(),
  body('name')
    .isLength({ min: 2, max: 50 })
    .withMessage('名前は2文字以上50文字以下である必要があります / Name must be between 2 and 50 characters')
    .trim()
    .escape(),
  body('password')
    .isLength({ min: 8 })
    .withMessage('パスワードは8文字以上である必要があります / Password must be at least 8 characters long')
    .matches(/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]/)
    .withMessage('パスワードは大小英字、数字、特殊文字を含む必要があります / Password must contain uppercase, lowercase, number, and special character')
];

// ユーザー作成エンドポイント / User Creation Endpoint
app.post('/api/users', createUserValidation, async (req, res) => {
  try {
    // バリデーションエラーチェック / Validation Error Check
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({
        error: 'バリデーションエラー / Validation Error',
        details: errors.array()
      });
    }

    const { email, name, password } = req.body;

    // 重複チェック / Duplicate Check
    const existingUser = await User.findOne({ email });
    if (existingUser) {
      return res.status(409).json({
        error: 'このメールアドレスは既に使用されています / Email address already in use'
      });
    }

    // パスワードハッシュ化 / Password Hashing
    const bcrypt = require('bcrypt');
    const saltRounds = 12;
    const hashedPassword = await bcrypt.hash(password, saltRounds);

    // ユーザー作成 / User Creation
    const newUser = new User({
      email,
      name,
      password: hashedPassword,
      createdAt: new Date(),
      isActive: true,
      role: 'user'
    });

    const savedUser = await newUser.save();

    // JWTトークン生成 / JWT Token Generation
    const token = jwt.sign(
      { 
        userId: savedUser._id, 
        email: savedUser.email, 
        role: savedUser.role 
      },
      process.env.JWT_SECRET,
      { expiresIn: '7d' }
    );

    // ログ記録 / Logging
    console.log(`新規ユーザー作成: ${email} / New user created: ${email}`);

    // レスポンス（パスワードを除外） / Response (exclude password)
    res.status(201).json({
      message: 'ユーザーが正常に作成されました / User created successfully',
      user: {
        id: savedUser._id,
        email: savedUser.email,
        name: savedUser.name,
        createdAt: savedUser.createdAt,
        role: savedUser.role
      },
      token
    });

  } catch (error) {
    console.error('ユーザー作成エラー / User creation error:', error);
    res.status(500).json({
      error: 'サーバー内部エラー / Internal server error',
      message: process.env.NODE_ENV === 'development' ? error.message : undefined
    });
  }
});

// ヘルスチェックエンドポイント / Health Check Endpoint
app.get('/api/health', (req, res) => {
  res.status(200).json({
    status: 'OK',
    timestamp: new Date().toISOString(),
    uptime: process.uptime(),
    environment: process.env.NODE_ENV || 'development'
  });
});

// エラーハンドリングミドルウェア / Error Handling Middleware
app.use((error, req, res, next) => {
  console.error('アプリケーションエラー / Application error:', error);
  res.status(error.status || 500).json({
    error: 'サーバーエラーが発生しました / Server error occurred',
    message: process.env.NODE_ENV === 'development' ? error.message : 'Internal server error'
  });
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
  console.log(`サーバーが起動しました / Server started on port ${PORT}`);
});

module.exports = app;

Core Web Vitals 最適化 / Core Web Vitals Optimization ▼

Googleの重要指標

Core Web Vitalsは、ユーザーエクスペリエンスの品質を測定する3つの重要な指標です：

  • LCP (Largest Contentful Paint): ページの主要コンテンツの読み込み時間 (目標: 2.5秒以下)
  • FID (First Input Delay): 最初のユーザー操作への応答時間 (目標: 100ms以下)
  • CLS (Cumulative Layout Shift): レイアウトの視覚的安定性 (目標: 0.1以下)

Google's Key Metrics

Core Web Vitals are three essential metrics that measure user experience quality:

  • LCP (Largest Contentful Paint): Loading time of main content (Goal: ≤2.5s)
  • FID (First Input Delay): Response time to first user interaction (Goal: ≤100ms)

function optimizeWebVitals() {
  // LCP 最適化 / LCP Optimization
  // 1. 画像の最適化 / Image optimization
  const images = document.querySelectorAll('img');
  images.forEach(img => {
    if (img.loading !== 'lazy') {
      img.loading = 'lazy';
      img.decoding = 'async';
    }
  });

  // 2. 重要なリソースのプリロード / Preload critical resources
  const criticalCSS = document.createElement('link');
  criticalCSS.rel = 'preload';
  criticalCSS.as = 'style';
  criticalCSS.href = '/critical.css';
  document.head.appendChild(criticalCSS);

  // FID 最適化 / FID Optimization
  // 3. 長いタスクの分割 / Split long tasks
  function yieldToMain() {
    return new Promise(resolve => {
      setTimeout(resolve, 0);
    });
  }

  async function processLargeArray(array) {
    const batchSize = 100;
    for (let i = 0; i < array.length; i += batchSize) {
      const batch = array.slice(i, i + batchSize);
      
      // バッチ処理 / Process batch
      batch.forEach(item => {
        // 処理ロジック / Processing logic
        processItem(item);
      });

      // メインスレッドに制御を戻す / Yield to main thread
      await yieldToMain();
    }
  }

  // CLS 最適化 / CLS Optimization
  // 4. 画像とiframeのサイズ指定 / Specify image and iframe dimensions
  const mediaElements = document.querySelectorAll('img, iframe, video');
  mediaElements.forEach(element => {
    if (!element.style.width && !element.style.height) {
      // アスペクト比を維持しながらサイズを指定 / Specify size while maintaining aspect ratio
      element.style.aspectRatio = '16 / 9'; // 例 / Example
      element.style.width = '100%';
    }
  });

  // 5. フォントの読み込み最適化 / Font loading optimization
  const fontLink = document.createElement('link');
  fontLink.rel = 'preload';
  fontLink.as = 'font';
  fontLink.type = 'font/woff2';
  fontLink.href = '/fonts/main-font.woff2';
  fontLink.crossOrigin = 'anonymous';
  document.head.appendChild(fontLink);
}

<!-- 3. 収益最適化のための遅延読み込み / Lazy Loading for Revenue Optimization -->
<script>
class AdSenseManager {
    constructor() {
        this.adsLoaded = new Set();
        this.intersectionObserver = null;
        this.adPerformance = new Map();
        this.init();
    }

    init() {
        // Intersection Observer で広告の遅延読み込み / Lazy load ads with Intersection Observer
        this.intersectionObserver = new IntersectionObserver(
            (entries) => this.handleIntersection(entries),
            {
                rootMargin: '200px', // 200px 手前で読み込み開始 / Start loading 200px ahead
                threshold: 0.1
            }
        );

        // パフォーマンス追跡 / Performance tracking
        this.trackAdPerformance();
        
        // レスポンシブ広告の動的調整 / Dynamic responsive ad adjustment
        this.handleResponsiveAds();
    }

    handleIntersection(entries) {
        entries.forEach(entry => {
            if (entry.isIntersecting) {
                const adElement = entry.target;
                const adId = adElement.dataset.adId;
                
                if (!this.adsLoaded.has(adId)) {
                    this.loadAd(adElement);
                    this.adsLoaded.add(adId);
                    this.intersectionObserver.unobserve(adElement);
                }
            }
        });
    }

    loadAd(adElement) {
        try {
            // 広告サイズの動的決定 / Dynamic ad size determination
            const containerWidth = adElement.offsetWidth;
            let adSize = this.getOptimalAdSize(containerWidth);
            
            // 広告スロットの設定 / Ad slot configuration
            adElement.innerHTML = `
                <ins class="adsbygoogle"
                     style="display:block"
                     data-ad-client="ca-pub-1184134440246706"
                     data-ad-slot="${adElement.dataset.adSlot}"
                     data-ad-format="${adSize.format}"
                     data-full-width-responsive="true"></ins>
            `;

            // AdSense の初期化 / AdSense initialization
            (adsbygoogle = window.adsbygoogle || []).push({});
            
            // パフォーマンス記録 / Performance recording
            this.recordAdLoad(adElement.dataset.adId);
            
        } catch (error) {
            console.error('AdSense loading error:', error);
            this.handleAdError(adElement, error);
        }
    }

    getOptimalAdSize(containerWidth) {
        // デバイスと画面サイズに基づく最適広告サイズの決定 / Optimal ad size based on device and screen size
        if (containerWidth <= 320) {
            return { format: 'auto', type: 'mobile-banner' };
        } else if (containerWidth <= 728) {
            return { format: 'auto', type: 'responsive' };
        } else {
            return { format: 'auto', type: 'leaderboard' };
        }
    }

    trackAdPerformance() {
        // 広告パフォーマンスの追跡 / Track ad performance
        const performanceObserver = new PerformanceObserver((list) => {
            list.getEntries().forEach((entry) => {
                if (entry.name.includes('googlesyndication.com')) {
                    this.recordAdMetrics(entry);
                }
            });
        });
        
        performanceObserver.observe({ entryTypes: ['resource', 'navigation'] });
    }

    recordAdMetrics(entry) {
        const metrics = {
            loadTime: entry.duration,
            startTime: entry.startTime,
            transferSize: entry.transferSize || 0,
            timestamp: Date.now()
        };
        
        // Google Analytics に送信 / Send to Google Analytics
        gtag('event', 'ad_performance', {
            event_category: 'AdSense',
            event_label: 'load_time',
            value: Math.round(metrics.loadTime),
            custom_parameter_1: entry.name
        });
    }

    handleResponsiveAds() {
        // 画面サイズ変更時の広告最適化 / Ad optimization on screen size change
        let resizeTimer;
        window.addEventListener('resize', () => {
            clearTimeout(resizeTimer);
            resizeTimer = setTimeout(() => {
                this.refreshResponsiveAds();
            }, 500);
        });
    }

    refreshResponsiveAds() {
        const ads = document.querySelectorAll('.adsbygoogle[data-responsive="true"]');
        ads.forEach(ad => {
            if (ad.dataset.adStatus === 'loaded') {
                // レスポンシブ広告の再最適化 / Re-optimize responsive ads
                (adsbygoogle = window.adsbygoogle || []).push({});
            }
        });
    }

    // A/B テスト用メソッド / A/B testing methods
    runAdPlacementTest() {
        const testVariant = Math.random() < 0.5 ? 'A' : 'B';
        
        // バリアント A: ヘッダー下部に広告配置 / Variant A: Place ad below header
        if (testVariant === 'A') {
            this.placeAdAfterElement('#header', 'header-ad');
        } 
        // バリアント B: 記事中央に広告配置 / Variant B: Place ad in article center
        else {
            this.placeAdAfterParagraph(3, 'content-ad');
        }

        // テスト結果の記録 / Record test results
        gtag('event', 'ad_test', {
            event_category: 'AdSense_AB_Test',
            event_label: `variant_${testVariant}`,
            custom_parameter_1: testVariant
        });
    }

    handleAdError(adElement, error) {
        // 広告読み込みエラーの処理 / Handle ad loading errors
        console.warn('AdSense Error:', error);
        
        // フォールバック広告の表示 / Display fallback ad
        adElement.innerHTML = `
            <div class="ad-fallback">
                <p>スポンサーリンク / Sponsored Link</p>
                <!-- 代替広告コンテンツ / Alternative ad content -->
            </div>
        `;

        // エラー追跡 / Error tracking
        gtag('event', 'ad_error', {
            event_category: 'AdSense',
            event_label: error.message,
            value: 1
        });
    }
}

// AdSense マネージャーの初期化 / Initialize AdSense Manager
document.addEventListener('DOMContentLoaded', () => {
    window.adSenseManager = new AdSenseManager();
    
    // 広告要素の観察開始 / Start observing ad elements
    document.querySelectorAll('.ad-container[data-ad-id]').forEach(ad => {
        window.adSenseManager.intersectionObserver.observe(ad);
    });
});
</script>
