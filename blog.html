台風で3連休引きこもり予定が…まさかの休日出勤

10月の3連休。天気予報を見たら台風直撃コース。「よし、これは引きこもって開発するしかない！」と意気込んでいました。

🌀 引きこもり計画
せっかくの連休だし、台風で外出できないなら、昨日ブログに書いた「日本酒サジェストアプリ」の開発を一気に進めようと計画。青森の地酒データベース構築を3日間でやりきる予定でした。

冷蔵庫にはカップ麺とレトルトカレーを大量にストック。コーヒー豆も挽きたて。モニターは2台並べて、片方にVSCode、片方にドキュメント。完璧な開発環境を整えて、台風が来るのを待っていました。

外は雨。家の中は暖かい。キーボードを叩く音だけが響く至福の時間…のはずだった。

📱 金曜夜の悪夢
金曜の夜10時。Slackに通知が来ました。「明日（土曜）、システム障害対応のため出社対応お願いします」…は？

人事システムのバッチ処理が深夜に異常終了したらしく、月曜の給与計算に間に合わせるため、土日で復旧作業が必要とのこと。しかも、リモートではなく「セキュリティ上、オンプレサーバーに直接アクセスが必要なので出社必須」。

台風のピークが土曜日なのに…。

🌧️ 台風の中の出勤
土曜朝7時。外は暴風雨。傘が意味をなさないレベル。ずぶ濡れになりながら駅まで歩き、運休ギリギリの電車に飛び乗って出社しました。

会社に着いたら、同じく呼び出された同僚が3人。みんな半泣き顔で「なんで俺たち…」と肩を落としていました。復旧作業は結局15時間かかり、土曜の夜11時までオフィスに缶詰。日付が変わる前にギリギリ帰宅。

日曜日は疲れ果てて1日中寝てました。月曜（祝日）も体調が戻らず、ほぼ寝て過ごす羽目に。

開発のモチベーションは残ってるけど、体力的にはボロボロ。次の週末こそ…と思いつつ、またSlackに怯える日々が始まります。

せめてもの救いは、台風で引きこもれなかった代わりに「ネタになった」こと。このブログ記事は、休日出勤の辛さを昇華した産物です（笑）。

新企画始動！日本酒サジェストアプリ「Sake Suggest」開発スタート

突然ですが、新しいプロジェクトを始めます。その名も「日本酒サジェストアプリ」！

📱 企画のきっかけ
日本酒って、全国に1万種類以上あるんですよ。地元の酒屋に行くと、棚いっぱいに銘柄が並んでいて、正直どれを選べばいいか分からない。好きな銘柄はあるけど、「これに近い味の別の日本酒ないかな？」って思っても、探すのが大変。

結局いつも同じ銘柄ばかり買ってしまう…。でも、せっかくなら新しい出会いも楽しみたい。そんな自分のために、「好きな銘柄に近い日本酒をサジェストしてくれるアプリ」を作ろうと思い立ちました。

🔍 既存サービスを調査してみた
まずは類似サービスがないか調べてみたところ、意外とたくさん見つかりました：

① Sakenomy（サケノミー） - 中田英寿氏監修。7,000種類以上の日本酒データベースで、ラベル認識や好み分析機能あり
② Sakenowa（さけのわ） - 飲んだ日本酒を記録＆レビュー。独自の味わい識別システムでレコメンド
③YUMMY SAKE - AIとブラインドテイスティングで好みを12タイプに分類
④ 日本酒の学校 - AIが類似銘柄をサジェスト。味のチャート調整も可能
⑤ SAKE Street 酒ナビ - 東大農学部と共同研究した機械学習モデルで好み診断

…めっちゃある（笑）。しかも、どれもAI・機械学習を使った本格派。正直、「もう作らなくていいんじゃ…？」と一瞬思いました。

💡 それでも作る理由
でも、既存アプリを使ってみて気づいたことがあります。多くは「アプリをダウンロード→会員登録→診断→レコメンド」という流れ。それはそれで良いんですが、「今すぐサクッと調べたい」時にはちょっと重い。

私が作りたいのは、もっとシンプルで軽量なWebアプリ。ブラウザで開いて、好きな銘柄を1つ入れるだけで、似た味わいの日本酒を5つ提案してくれる。アプリDL不要、会員登録不要、PWA対応でオフラインでも使える。

🛠️ 技術的なアプローチ
データソースは、SAKETIMEやSakenomyの公開データ、各蔵元の公式情報を活用予定。味わいの類似度計算には、ベクトル空間モデルを使って「甘辛度」「香り」「コク」などのパラメータで距離を測定します。

機械学習は使わず、あくまでシンプルなロジックで。データベースも軽量化して、1,000銘柄程度に絞り込む予定です。

📅 今後の展開
まずは青森の地酒データベースから構築スタート。地元の酒造データを手作業で整備しながら、サジェストアルゴリズムを検証していきます。完成したら、このブログで公開しますのでお楽しみに！

冷蔵庫の余り物レシピアプリの作り方｜食材ロス削減とプログラミング学習を両立

🌱 食材ロス問題とWebアプリ開発
日本では年間約612万トンの食品ロスが発生しており、その約半分は家庭から出ているというデータがあります。「冷蔵庫の奥で野菜が腐っていた」「賞味期限切れの調味料が出てきた」…誰もが経験したことがあるのではないでしょうか。

この課題を解決するために、私は冷蔵庫の余り物からレシピを提案するWebアプリ「CookSuggest」を開発しました。このアプリを作る過程で学んだ技術と、プログラミング初心者でも実践できる開発のコツを紹介します。

💡 アプリの仕組み｜JavaScriptで実現するレシピ検索
CookSuggestは、以下のシンプルな仕組みで動作します：
1. ユーザーが「玉ねぎ、豚肉、醤油」などの食材を入力
2. JavaScriptで1,000種類以上のレシピデータベースを検索
3. 入力された食材を多く含むレシピをスコアリング
4. 3秒以内に5つのおすすめレシピを表示

データベースはJSON形式で、外部サーバー不要。すべてブラウザ内で完結するため、高速かつプライバシーにも配慮した設計です。
👉 CookSuggestを試してみる

📚 初心者でも作れる！使用技術スタック
このアプリで使用した技術は、すべて無料で学べる基本的なものです：

  • HTML/CSS - レスポンシブデザイン、Flexbox、CSS Grid
  • JavaScript - 配列操作（filter, map, sort）、ローカルストレージ
  • PWA - Service Worker、マニフェストファイル（オフライン対応）
  • GitHub Pages - 無料ホスティング

プログラミング学習3ヶ月目でも、十分実装可能なレベルです。重要なのは「解決したい課題」を明確にすること。私の場合は「冷蔵庫の余り物を使い切りたい」という自分自身の悩みがモチベーションでした。

🔍 レシピデータの集め方｜スクレイピングとデータ整形
1,000件のレシピデータは、Pythonでクックパッド風のレシピサイトからスクレイピングしました。ただし注意点として：
・robots.txtの確認必須
・アクセス頻度の制限（1秒に1リクエスト以下）
・著作権に配慮し、レシピ名と材料のみ取得

データはCSV形式で保存し、JavaScriptで読み込めるJSONに変換。この工程でR言語のTidyverseが大活躍しました。データクレンジング、重複削除、カテゴリ分類など、効率的に処理できます。

🚀 実装で工夫したポイント
1. 曖昧検索機能
「たまねぎ」「玉ねぎ」「タマネギ」すべてに対応するため、ひらがな変換とキーワード正規化を実装。ユーザー体験が格段に向上しました。

2. スコアリングアルゴリズム
単純な材料マッチングではなく、「手持ち食材でどれだけ作れるか」を計算。足りない材料が少ないレシピを優先表示します。

3. PWA化でオフライン対応
Service Workerを使い、一度アクセスすればオフラインでも利用可能に。キャンプや旅行先でも使えると好評です。

📈 実際の効果｜食材ロス削減とスキルアップ
このアプリを3ヶ月使った結果：
✅ 食材ロスが約60%削減（週に捨てる野菜が5個→2個）
✅ 食費が月5,000円削減
✅ プログラミングスキルが実践で向上
✅ GitHubのコントリビューションが活発化し、転職活動でアピールできた

🌟 これからWebアプリ開発を始める方へ
「プログラミング初心者には無理」と思うかもしれませんが、実は小さな不便を解決するアプリこそ、初心者に最適な題材です。理由は3つ：
1. 自分が使うから、要件定義が明確
2. 小規模なので、3ヶ月で完成可能
3. 実際に使うから、継続的な改善モチベーションが湧く

私も人事部門の普通のサラリーマンで、プログラミング経験ゼロからスタート。R言語でのデータ分析から始め、Web開発に広げました。独学で十分です。

非エンジニアがWebベースのコンテンツを作ってみる

30代サラリーマンが働きながらプログラミング学習｜挫折しないための5つの秘訣

⏰ 時間がない社会人の現実
30代・人事部門勤務の私が、プログラミング学習を始めたのは2年前。当時の状況：残業月80時間、休日出勤あり、帰宅は22時。「時間がない」が口癖でした。

それでも、今ではWebアプリを10個以上開発し、R言語・Python・JavaScriptを実務レベルで使えるようになりました。どうやって時間を作り、挫折せずに続けたか？5つの秘訣をお伝えします。

📌 秘訣1：完璧主義を捨てる「80点主義」
最大の敵は「ちゃんと理解してから次に進む」という完璧主義。書籍を1ページずつ理解しようとすると、途中で嫌になります。

私のルール：「80点理解したら次へ」。わからない部分はメモして後回し。実際にコードを書きながら戻ると、不思議と理解できます。

📌 秘訣2：朝活で「1日30分」の習慣化
夜は疲れて集中できない。そこで、朝6時に起きて30分だけコードを書く習慣を作りました。

朝の30分は、夜の2時間に匹敵する集中力。小さな達成感でスタートする1日は、驚くほどポジティブになります。

ポイント
・スマホを寝室に持ち込まない（SNSの誘惑を断つ）
・前夜に「明日やること」を決めておく
・コーヒーとお気に入りの音楽でルーティン化

📌 秘訣3：通勤時間を「インプットタイム」に
片道40分の通勤時間、往復で80分。これを学習時間に変換しました。

電車内でやったこと
・Udemyの動画講座（1.5倍速）
・技術ブログ記事を読む（Qiita、Zenn）
・Progateのスマホアプリでコーディング練習
・Podcastで英語の技術番組（Syntax.fm、JS Party）

月20日出勤なら、月26時間。年間312時間の学習時間を確保できました。

📌 秘訣4：「作りたいもの」から逆算学習
書籍を最初から順番に読む学習法は、社会人には向きません。時間がかかりすぎて、途中で飽きます。

私の方法：「作りたいアプリ」を先に決めて、必要な知識だけを学ぶ。

例：料理アプリを作りたい
1. 必要な技術をリストアップ（HTML/CSS、JavaScript、レシピデータ）
2. 各技術の「最低限必要な部分」だけ学ぶ
3. まず動くものを作る（見た目は後回し）
4. 動いてから、細部を改善

この方法なら、最短1週間で「動くアプリ」が完成。達成感が次のモチベーションになります。

📌 秘訣5：SNSで学習記録を公開する
Twitterで「#今日の積み上げ」ハッシュタグで、毎日の学習内容をツイート。これが驚くほど効果的でした。

メリット
・公言することで「やらなきゃ」プレッシャー
・同じ学習者とのつながり（励まし合い）
・「いいね」の承認欲求が継続のモチベーション
・自分の成長記録が可視化される

3ヶ月続けると、フォロワーが増え、「応援してくれる人」ができます。この存在が、挫折防止に絶大な効果。

⚠️ やってはいけないNG行動
1. 複数言語を同時に学ぶ - 混乱して全部中途半端に
2. 高額スクールに頼りすぎ - 受け身では身につかない
3. エラーをすぐ質問 - 自分で調べる力が育たない
4. 最新技術を追いかける - 基礎がないと理解できない
5. 完成度にこだわりすぎ - 公開しないと成長しない

社会保険労務士試験合格への道｜働きながら一発合格した勉強法とスケジュール

📖 なぜ社労士を目指したのか
人事部門で5年働く中で、「専門知識の不足」を痛感する場面が増えました。労働法、社会保険、年金制度…制度改正のたびに外部の社労士に頼る状況に、「自分でもっと詳しくなりたい」と思ったのがきっかけです。

📊 社労士試験の難易度と合格率
社会保険労務士試験の合格率は約6-7％。司法試験ほどではないものの、難関国家資格の一つです。

試験科目（全8科目）
1. 労働基準法及び労働安全衛生法
2. 労働者災害補償保険法
3. 雇用保険法
4. 労務管理その他の労働に関する一般常識
5. 社会保険に関する一般常識
6. 健康保険法
7. 厚生年金保険法
8. 国民年金法

各科目で基準点（約4割）をクリアし、かつ総得点で上位7％に入る必要があります。一つでも科目落ちすると不合格。バランスが重要です。

⏰ 私の学習スケジュール（10ヶ月・総学習時間1000時間）
【第1期】基礎固め（3ヶ月・300時間）
・平日：朝1時間、夜2時間
・休日：5-6時間
・教材：TAC通信講座、基本テキスト精読
・目標：全科目を一通り理解

この時期は「完璧理解」を目指さない。60%理解で次へ進むイメージ。年金科目は特に難解ですが、後で何度も戻るので深追い禁止。

【第2期】過去問演習（4ヶ月・400時間）
・過去10年分の過去問を3周
・間違えた問題はノートに記録
・横断整理（科目をまたいだ知識の統合）
・模試を月1回受験

ここが最も重要な時期。過去問を解くことで「出題パターン」が見えてきます。特に選択式問題（穴埋め）は、頻出キーワードが決まっています。

【第3期】直前期（3ヶ月・300時間）
・法改正情報の徹底チェック
・苦手科目の集中学習
・模試を週1回受験（計12回）
・暗記カードで数字の暗記

試験1ヶ月前は、新しいことは学ばず、既存知識の反復のみ。特に「数字」（保険料率、給付額、期間）は直前まで記憶が抜けるので、毎日復習。

・講義動画（1.5倍速で視聴）

💡 合格のための5つのコツ
1. 「捨て科目」を作らない
基準点制度があるため、得意科目で満点取っても、1科目でも基準点割れすると不合格。苦手科目こそ優先的に学習。

2. 横断学習で知識を統合
例えば「出産」関連の給付は、健康保険・雇用保険・労災保険すべてに出てきます。科目ごとではなく、テーマごとに整理すると記憶に定着。

3. 法改正情報は命
試験の約20%は、直近2年の法改正から出題。TACの「法改正まとめ冊子」は必読。

4. 模試は本番と同じ時間に受ける
試験は午前3時間20分・午後3時間10分の長丁場。集中力・体力が必要。模試で本番のシミュレーションを何度も経験。

5. 選択式対策を軽視しない
選択式（穴埋め）は運要素が大きく、ここで不合格になる人が多い。過去問だけでなく、予想問題集も解く。

😰 挫折しそうになったとき
正直、何度も「もう無理」と思いました。特に厚生年金・国民年金の計算問題は、理解に3ヶ月かかりました。

乗り越えた方法
・学習仲間を作る（SNSで「#社労士受験」仲間と励まし合い）
・小さな目標設定（今日は労基法10ページ、など）
・ご褒美制度（1科目終わったら好きなもの食べる）
・合格後のビジョンを明確化（資格手当月3万円、昇進）

🎉 合格発表の瞬間
11月の合格発表日、官報で自分の受験番号を見つけた瞬間は、今でも忘れられません。10ヶ月の努力が報われた瞬間でした。

合格後、社内で「労務管理のスペシャリスト」として認められ、給与も昇格。何より、自信がつきました。

無料で作れるWebアプリ完全ガイド｜初心者が0円でアプリ公開する方法

💰 お金をかけずにWebアプリを作りたい
「Webアプリを作りたいけど、サーバー代が…」と諦めていませんか？実は、完全無料でアプリを開発・公開できる時代です。私も初期費用0円で、10個以上のアプリを公開しています。

この記事では、無料ツールだけでWebアプリを作る方法を、初心者向けに徹底解説します。

🛠️ 必要な無料ツール一覧
1. コードエディタ：VS Code（無料）
Microsoftが提供する最強エディタ。プログラマーの90%以上が使用。拡張機能が豊富で、HTML/CSS/JavaScriptのコード補完が優秀。

2. ホスティング：GitHub Pages（無料）
静的サイト（HTML/CSS/JavaScript）なら、GitHubで無料ホスティング。独自ドメインも設定可能（ドメイン代のみ年1000円程度）。

3. バックエンド：Supabase / Firebase（無料枠あり）
データベース、認証、ストレージが無料で使える。小規模アプリなら無料枠で十分。

4. 画像素材：Unsplash / Pexels（完全無料）
商用利用可能な高品質写真が無料。クレジット表記も不要。

5. アイコン：Font Awesome（無料）
Webアイコンの定番。CDNで読み込めば即使用可。

📝 開発の流れ（5ステップ）
Step 1: アイデア出し
「自分が欲しいアプリ」から考える。例：
・冷蔵庫の余り物レシピ検索
・地震情報の通知
・勉強時間の記録ツール

大規模でなくてOK。まずは「1つの機能」に絞る。

Step 2: 技術選定
初心者におすすめ：
・フロントエンド：HTML + CSS + JavaScript（バニラJS）
・バックエンド：なし（ローカルストレージで保存）
・デプロイ：GitHub Pages

React/Vueは後回し。まずは基礎を固める。

Step 3: プロトタイプ作成
デザインは後回し。まず「動くもの」を作る。

最小限のHTMLテンプレート
```html
<!DOCTYPE html>
<html>
<head>
<title>アプリ名</title>
</head>
<body>
<h1>Hello World</h1>
<script src="app.js"></script>
</body>
</html>
```

これだけで、ブラウザで表示できます。

Step 4: GitHub Pagesで公開
1. GitHubアカウント作成（無料）
2. リポジトリ作成（例：my-recipe-app）
3. index.htmlをアップロード
4. Settings → Pages → Sourceを"main branch"に設定
5. 5分後、https://ユーザー名.github.io/my-recipe-app/ で公開

これだけで、世界中からアクセス可能なアプリが完成！

Step 5: 改善を繰り返す
・CSSで見た目を整える
・JavaScriptで機能追加
・ユーザーからフィードバック収集
・バグ修正

🎨 デザインのコツ（センス不要）
「デザインセンスがない」と悩む必要はありません。既存の優れたデザインを参考にすればOK。

参考にすべきサイト
・Dribbble（デザインギャラリー）
・Tailwind CSS（簡単に綺麗なデザイン）
・Bootstrap（定番CSSフレームワーク）

色選びは「Coolors」で自動生成。フォントは「Google Fonts」で無料。
