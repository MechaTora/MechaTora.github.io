# サブドメイン別 やることリスト（AdSense/品質観点）

最終監査日: 2026-07-23 ／ 対象: kimochi・odai・sharonabi・x-algo（すべて別リポジトリ）

各サイト共通で「AdSense申請に必要な土台」＝ ①独立したプライバシーポリシーページ、②お問い合わせページ（フォーム or 連絡手段）、③運営者情報（About）、④GA4タグ、⑤十分な独自コンテンツ。以下は共通土台に加えた、サイト個別の課題です。

## 1. kimochi.mechatora.com（仕事の気持ちメータ）

**現状**：動作良好。地図（Leaflet）・全国統計・FAQ・「プライバシーへの配慮」説明文まであり、完成度は高い。

**やるべきこと**
1. **シェア投稿の旧ドメイン混入を修正**（最優先）
   - シェアフィードやX投稿テンプレートに旧NetlifyのURL `https://voluble-fudge-bb2bc6.netlify.app/` が残っている。
   - 投稿生成箇所を `https://kimochi.mechatora.com/` に一括置換。過去投稿の表示分も可能なら差し替え。
2. **法務ページを独立ページ化**
   - 本文中の「プライバシーへの配慮」は良いが、AdSense的には独立した `/privacy-policy` ページが必要。
   - お問い合わせ導線（メール or フォーム or mechatora.com/contact への誘導）を設置。
3. **広告方式の整理**
   - 現状は「※広告は外部サイトへのリンクです」型（外部リンク広告）。AdSenseに一本化するなら、外部リンク広告を撤去し AdSense タグへ差し替え。
4. **ブランド導線**：フッターに「メカとら（mechatora.com）」へのリンクを追加し、姉妹サイト感を出す。
5. GA4（G-6NJKWS2RJ8 とは別 or 同一か方針決め）と favicon を本体と統一。

## 2. odai.mechatora.com（Note お題制限特定ナビ）

**現状**：説明・FAQは整っているが、トップの「Noteからお題タグを取得中…」がスピナーのまま＝**主機能の自動取得が動いていない可能性**。

**やるべきこと**
1. **お題タグ自動取得の不具合を切り分け（最優先・機能面）**
   - 「Noteからお題タグを取得中…」で止まる原因を確認（Note側のRSS/APIの仕様変更、CORS、取得先URLの失効、非同期エラーなど）。
   - ブラウザのDevToolsのConsole/Networkでエラーを確認 → 取得先の修正 or フォールバック（手動更新ボタンは動くか）を実装。
   - 取得に失敗したときは「一覧を取得できませんでした（時間をおいて再試行）」と**明示表示**する（無限スピナーを避ける）。
2. **空のスポンサー枠の扱い**：広告が入っていない「スポンサーリンク」枠が複数ある。AdSenseに切り替えるか、当面は枠を非表示に。
3. **法務・About・お問い合わせページ**を追加（共通土台）。
4. **コンテンツ増強**：ツール＋説明＋FAQのみでやや薄い。「お題タグ活用術」「Noteコンテスト攻略」等の記事を数本足すと審査に有利。
5. ブランド導線（mechatora.com）とfavicon・GA4を統一。

## 3. sharonabi.mechatora.com（シャロナビ｜社労士ポータル）

**現状**：デザイン・作り込みは4サイト中トップ。ただし **第三者コンテンツのRSS全文転載が多い**（『資格の大原』ブログ、個人ブログ「シャロ勉」、社労士試験公式ニュース等）。

**やるべきこと**
1. **転載コンテンツの是正（AdSense最重要）**
   - 他者ブログ/ニュースの**全文転載は不可**。AdSenseの「価値の低いコンテンツ／著作権」で落ちる典型。
   - 方針: (a) 全文表示をやめ「タイトル＋数行の要約＋出典元リンク」に留める（リンク集/キュレーション型に）／(b) もしくは自分の言葉での**オリジナル解説記事**へ差し替える。
   - 「first appeared on 『資格の大原』ブログ」等のRSSフッター文言が出ている箇所は特に要注意。
2. **一次コンテンツ（オリジナル）を主役に**
   - 「合格体験記」「試験ガイド」「予備校比較」など自作コンテンツを前面に。集約系は補助的な位置づけへ。
3. **YouTube**は埋め込み（iframe）ならOK。サムネ転載＋自前ホストは避ける。
4. **法務・お問い合わせ**：「管理人について」はあるので、プライバシーポリシー／お問い合わせページの有無を確認し、無ければ追加。
5. ニュースの鮮度と出典明記（公式へのリンク）を徹底。
6. ブランド導線・favicon・GA4を統一。

> メモ: これはかつて本体から外した「社会保険ニュース（自動転載）」と同種のリスク。AdSense申請前に必ず解消したいポイント。

## 4. x-algo.mechatora.com（Algorithm Watch）

**現状**：デザインは良いが **記事が1本のみ**＝極端に薄く、現状ではAdSense不可レベル。

**やるべきこと**
1. **コンテンツを大幅に増やす（最優先）**
   - 最低でも記事10本以上を目安に。Xアルゴリズム（Phoenixエンジン、推薦システム、オープンソース公開）の解説・変更履歴・技術深掘りを継続追加。
   - GitHub（xai-org/x-algorithm）のコミットを**そのまま貼るのではなく**、自分の言葉で解説する形に（転載回避＋独自価値）。
2. **更新の継続性**：単発で終わらせず、アルゴリズム更新を追跡して定期更新する運用に（サイトの趣旨＝Watchなので相性良い）。
3. **法務・About・お問い合わせページ**を追加（共通土台）。
4. ブランド導線・favicon・GA4を統一。
5. 記事が揃うまではAdSense申請を保留（薄いまま出すと不承認履歴が残る）。
